# Инструкция конструктора runtime-symbols шаблонов.

1. Будет питоновский конструктор, который будет брать список символов заданных в CFG/app.json -> "symbols".
2. Опираясь на шаблон CFG/temp/_base.json, создает пер символьный кеш типа btcusdt.json (в зависимости от конкретного символа), который будет лежать по пути CFG/runtime/...
3. В его стркутуру нужно добавить флаг состояния active false.
4. Это правило применяется к каждому символу в списке symbols но при условии что файл не сгенерирован.
5. Остальная часть управkения и редактирования состояний этих фалов будет прописано в других инструкциях кода.
6. При каждом запуске бота должен срабатывать input с напоминанием проверить настройки и выбором продолжить или скипнуть запуск. (также эта опция должна включаться отключаться флагом в CFG/app.app.json -- добавишь семантический.)
7. Пример пер символьного шаблона рантайм смотри в CFG/runtime/example.json. (там еще дополнительные флаги и состояния)
